# Day08 智能数字媒体内容审核系统验收清单

## 1. 使用说明

本清单是产品负责人、测试人员和现场验收教师共同使用的验收基线。所有复选框默认未勾选，表示“尚未用真实运行证据确认”，不代表功能失败。只有实际执行步骤并保存证据后，才能把条目标记为完成。

### 1.1 验收环境

- 操作系统：Windows；
- Python 环境：`conda activate yolo`；
- 服务地址：`http://127.0.0.1:7880`；
- 浏览器：Chrome 或 Edge 桌面版；
- 模型位置：`models/yolo11n.pt` 或团队自训练模型的实际配置路径；
- 输出位置：`outputs/<job_id>/`；
- 视频工具：`ffmpeg` 与 `ffprobe` 命令可执行。

### 1.2 证据要求

每条验收记录至少关联一种证据：

- API 响应截图或保存的 JSON；
- 页面操作截图；
- `job.json`、`analysis_report.json` 或证据图片；
- 自动化测试输出；
- 对应 Git commit；
- Bug 复现和回归记录。

### 1.3 通过规则

1. REQ-001 至 REQ-009 为 P0，任一项存在阻断问题时项目不能认定为完成。
2. REQ-010 为本组承诺的 P1 加分功能，应在 P0 通过后验收。
3. 预期为 4xx 或 failed 的异常场景，只要响应、任务状态和错误说明符合要求，应判定测试通过。
4. 实际结果与预期不一致时不得勾选，必须在 `docs/BUG_RECORD.md` 中记录并完成回归。
5. 测试报告最终至少包含 5 条正常、5 条异常和 2 个真实 Bug 的解决过程。

- [ ] 完成任务生成可读取的 `job.json` 和 `analysis_report.json`。
- [ ] 页面能展示检测结果、机器结论和证据。

## 3. 按需求验收

### AC-001 / REQ-001 上传素材并创建任务

**前置条件：** 服务运行正常；准备一张非空 `.jpg` 图片。

- [ ] 在新建任务弹窗输入项目名称并选择图片，点击提交。
- [ ] 请求使用 `multipart/form-data`，提交期间按钮防止重复点击。
- [ ] 接口返回 HTTP 201、`ok: true`、非空 job_id 和任务数据。
- [ ] 页面立即展示任务编号，而不是等待 YOLO 推理结束。
- [ ] `outputs/<job_id>/input/` 中存在上传文件。
- [ ] `job.json` 中的 project_name、asset_name、created_at 和 status 正确。

**预期证据：** 创建响应 JSON、任务列表截图、input 目录和 job.json。

### AC-002 / REQ-002 异步处理与状态查询

**前置条件：** AC-001 创建成功。

- [ ] 触发分析后状态从 created 进入 queued，再进入 running。
- [ ] 前端轮询任务详情，并在状态结束后停止轮询。
- [ ] 正常任务最终进入 completed，started_at 和 completed_at 均非空。
- [ ] 人为触发处理异常时任务进入 failed，并记录 completed_at 和 error。
- [ ] created、queued、running 任务不能进行人工复核或删除。
- [ ] 非法状态转换返回错误，原 job.json 仍为合法 JSON。

**预期证据：** 不同状态响应、页面状态截图、完成和失败 job.json。

### AC-003 / REQ-003 图片和视频 YOLO 检测

**前置条件：** 模型文件存在；准备合法图片和视频。

- [ ] 健康检查的 model_ready 为 true，模型名称与实际文件一致。
- [ ] 图片任务执行真实 YOLO 推理，不使用固定或伪造结果。
- [ ] 每条图片检测包含 class_name、confidence 和四个 bbox 坐标。
- [ ] 视频任务通过 ffprobe 获取时长、分辨率、编码和音频轨道信息。
- [ ] OpenCV 按任务规则中的采样间隔读取视频帧。
- [ ] 视频帧结果包含 frame_idx、time_sec 和 detections。
- [ ] confidence 位于 0 至 1，bbox 不超出图像边界。

**预期证据：** 模型日志、图片/视频报告、媒体信息和采样帧文件。

### AC-004 / REQ-004 配置化审核规则与结论

**前置条件：** 准备可控检测数据或稳定测试素材。

- [ ] 风险类别置信度为 0.60 时结论为 reject。
- [ ] 风险类别置信度为 0.35 时结论为 review。
- [ ] 风险类别置信度低于 0.35 时不单独触发审核结论。
- [ ] 警告类别置信度不低于 0.35 且无 reject 命中时结论为 review。
- [ ] 没有触发风险或警告规则且处理正常时结论为 pass。
- [ ] 同时命中 reject 和 review 条件时最终结论为 reject。
- [ ] 报告保存命中类别、置信度、规则阈值和可读原因。

**预期证据：** 三种结论报告、阈值边界测试输出。

### AC-005 / REQ-005 证据帧与审核报告留档

**前置条件：** 完成一个包含风险命中的分析任务。

- [ ] keyframes 目录中至少存在规则要求数量的证据图片。
- [ ] 证据图包含正确的类别、置信度和检测框。
- [ ] analysis_report.json 是合法 UTF-8 JSON。
- [ ] 报告包含任务信息、规则快照、检测结果、机器结论、原因和证据路径。
- [ ] job.json.result_file 正确指向分析报告。
- [ ] 报告引用的所有证据文件真实存在并可由页面访问。
- [ ] 零命中任务明确记录空检测和 pass 原因，不伪造证据。

**预期证据：** 报告文件、证据图片、报告接口响应。

### AC-006 / REQ-006 审核结果可视化

**前置条件：** 至少准备一个 completed、一个 failed 和一个无检测结果任务。

- [ ] 任务列表显示编号、项目名称、素材、状态、创建时间和结论。
- [ ] 审核工作台显示创建、开始和结束时间。
- [ ] 完成任务显示机器结论、原因、检测明细和证据图片。
- [ ] 通过、待复核、不通过使用一致的文字和视觉颜色。
- [ ] 加载中、空结果、完成和失败分别有专门状态视图。
- [ ] API 错误在页面可见区域提示，不只输出到浏览器控制台。
- [ ] 返回任务列表后页面状态正常，不残留上一任务数据。

**预期证据：** 各状态页面截图和对应报告 JSON。

### AC-007 / REQ-007 人工复核并写回结果

**前置条件：** completed 任务报告存在。

- [ ] 审核员可在 pass、review、reject 中选择人工结论。
- [ ] 可填写审核备注和复核人，并提交复核。
- [ ] 接口返回 `ok: true`，页面立即展示人工最终结论。
- [ ] 报告新增 manual_conclusion、review_note、reviewer 和 reviewed_at。
- [ ] 服务重启并重新打开任务后，人工复核内容保持不变。
- [ ] machine_conclusion、机器原因和原始证据未被覆盖。
- [ ] 非法结论或非 completed 任务的复核请求被明确拒绝。

**预期证据：** 复核前后报告对比、页面截图、PATCH 响应。

### AC-008 / REQ-008 历史任务重开与删除

**前置条件：** outputs 中存在多种状态任务。

- [ ] 任务列表按 created_at 倒序排列。
- [ ] 服务重启后仍能列出并重新打开历史任务。
- [ ] 重开任务的数据来自 JSON，与之前保存的报告一致。
- [ ] completed 或 failed 任务可经确认后删除。
- [ ] 删除成功后目标任务目录和列表项消失。
- [ ] created、queued、running 任务的删除请求返回冲突错误。
- [ ] 未知或含路径字符的 job_id 不会访问或删除 outputs 外部内容。

**预期证据：** 重启前后截图、删除响应、目录对比和路径安全测试。

### AC-009 / REQ-009 异常输入和失败提示

**前置条件：** 准备错误格式、空文件和不可解码素材；准备模型缺失配置。

- [ ] 所有失败 API 响应包含 `ok: false` 和非空 error。
- [ ] 参数或素材错误使用合适的 4xx 状态码。
- [ ] 模型缺失、ffprobe 失败、OpenCV 解码失败或推理异常使任务进入 failed。
- [ ] failed 任务保留 job.json、completed_at 和 error。
- [ ] 前端在表单或任务详情中显示可读错误。
- [ ] 一个任务失败后 `/api/health` 和其他任务查询仍可使用。
- [ ] 接口参数错误不会导致 Flask 服务退出。

**预期证据：** 异常响应、失败 job.json、错误页面截图、健康检查结果。

### AC-010 / REQ-010 页面维护审核规则

**前置条件：** P0 闭环已通过；打开规则配置弹窗。

- [ ] 页面可编辑风险类别、警告类别、拒绝阈值和复核阈值。
- [ ] 页面可编辑最小证据帧数和视频采样间隔。
- [ ] 前后端均校验 `0 <= review_confidence < reject_confidence <= 1`。
- [ ] 证据帧数小于 1 或采样间隔不大于 0 时拒绝保存并解释原因。
- [ ] 保存合法规则后新任务使用新规则快照。
- [ ] 修改全局规则不改变已有任务报告中的规则。
- [ ] 恢复默认后配置与 default_rules.json 一致。

**预期证据：** 规则页面截图、新旧任务 settings 对比和非法输入提示。

## 6. API 验收

| 完成 | 方法 | 路径 | 成功检查 | 失败检查 |
|---|---|---|---|---|
| [ ] | GET | `/api/health` | HTTP 200，状态与模型信息准确 | 模型缺失时 model_ready 为 false |
| [ ] | POST | `/api/jobs` | HTTP 201，返回 ok、job_id 和任务 | 参数、格式、空文件返回 4xx |
| [ ] | GET | `/api/jobs` | HTTP 200，返回任务数组 | 无任务时返回空数组而非错误 |
| [ ] | GET | `/api/jobs/<job_id>` | HTTP 200，返回完整任务 | 未知 id 返回 404 |
| [ ] | DELETE | `/api/jobs/<job_id>` | 已结束任务删除成功 | 运行中任务返回 409，非法 id 被拒绝 |
| [ ] | POST | `/api/jobs/<job_id>/analyze` | 接受分析并快速返回 | 未知/错误状态任务被拒绝 |
| [ ] | PATCH | `/api/jobs/<job_id>/review` | 人工复核写回成功 | 非法结论、未完成任务被拒绝 |
| [ ] | GET | `/api/jobs/<job_id>/report` | 返回合法报告 JSON | 报告不存在时返回明确错误 |

接口通用检查：

- [ ] 成功业务响应包含 `ok: true`。
- [ ] 失败业务响应包含 `ok: false` 和可读 error。
- [ ] Content-Type 与响应内容一致。
- [ ] 请求参数错误不会产生未处理的 500 或导致进程退出。

## 11. 方向适用性说明

本项目选择方向 A，因此任务书中仅针对方向 B 的推荐片段边界、粗剪视频、音频保留和多发布规格验收不适用。方向 A 仍需支持视频上传、FFmpeg 媒体探测、OpenCV 采样、YOLO 检测和证据帧留档。
